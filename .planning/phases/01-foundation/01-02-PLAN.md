**Replace src/app/layout.tsx** with exactly:

```tsx
import type { Metadata } from "next";
import { Inter, JetBrains_Mono } from "next/font/google";
import "./globals.css";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
});

const jetbrainsMono = JetBrains_Mono({
  subsets: ["latin"],
  variable: "--font-jetbrains-mono",
});

export const metadata: Metadata = {
  title: "boilerplate-vercel-neon",
  description: "Next.js + Neon Auth + Vercel boilerplate",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html
      lang="en"
      className={`${inter.variable} ${jetbrainsMono.variable} h-full antialiased`}
    >
      <body className="min-h-full flex flex-col font-sans">{children}</body>
    </html>
  );
}
```

Key points:
- Both fonts use the `variable` option (injects CSS custom property) — NOT `className` — so both variables are available globally
- `inter.variable` injects `--font-inter`, `jetbrainsMono.variable` injects `--font-jetbrains-mono`
- The `html` element receives both variable classes
- `font-sans` on body uses Tailwind's default sans stack (globals.css body rule overrides with var(--font-inter))

**Replace next.config.ts** with exactly:

The `env` block injects APP_VERSION at build time using the version from package.json (via npm_package_version env var set by Node/pnpm during build). This makes `process.env.APP_VERSION` available in client and server code.
