  <action>
Create `.github/workflows/ci.yml` with exactly this content. Create `.github/workflows/` directory first if it does not exist.

```yaml
name: CI

on:
  push:
    branches:
      - main
      - dev
      - staging
  pull_request:
    branches:
      - main

env:
  HUSKY: 0

jobs:
  ci:
    name: CI Pipeline
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
          persist-credentials: false

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm lint

      - name: Type check
        run: pnpm type-check

      - name: Build
        run: pnpm build

      - name: Unit tests
        run: pnpm test:unit

      - name: Install Playwright browsers
        run: npx playwright install chromium --with-deps

      - name: E2E tests
        run: pnpm test:e2e

      - name: Semantic Release
        if: github.event_name == 'push'
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: npx semantic-release
```

Key decisions reflected in this file:
- `fetch-depth: 0` — Semantic Release needs full git history to determine next version
- `persist-credentials: false` — Required by @semantic-release/git to push the release commit back to the repo using GITHUB_TOKEN
- `HUSKY: 0` — Prevents Husky from installing hooks during `pnpm install` in CI
- `pnpm/action-setup@v4` with `version: 10` — Matches pnpm@10.33.0 constraint in package.json engines
- `cache: 'pnpm'` on setup-node — Caches node_modules based on pnpm-lock.yaml hash
- `--frozen-lockfile` — Fails if pnpm-lock.yaml is out of sync (enforces lockfile integrity in CI)
- `npx playwright install chromium --with-deps` — Installs chromium and its OS-level dependencies (fonts, libs) needed on ubuntu-latest
- Semantic Release step: `if: github.event_name == 'push'` — Skips release on PRs (per D-05, D-22)
- Single job (not split into multiple jobs) — Simpler, no artifact passing needed, fast enough for a boilerplate
